You have been invoked to personalize this Astro static web page for a new client!

This project is a base presentation template designed to be customized manually by you (the AI) for each new business. 

The user will provide you with two sets of raw materials in the `assets/` folder. The text file was generated by an external AI performing a deep research of the location.
1. `assets/businessData.md`: A text file where the user has pasted the output of the deep research, containing all the copywriting, color palettes, fonts, history, location details, and contact info for the venue.
2. `assets/images/`: A folder containing all the high-quality images of the venue and its products.

Your goal is to consume these two raw materials to implement the final website, replacing the generic defaults in this template with the specific information and assets provided for this client.

Follow these steps exactly to execute the `/implement-web-page` command:

1. **Analyze the Materials**
   - Read `assets/businessData.md` to deeply understand the business's identity, copy, contact info, and vibe.
   - Look inside the `assets/images/` directory and analyze all available images to understand the visual aesthetic and content you are working with.

2. **Personalize the Astro Components & CSS (Code Replacement)**
   - You must evaluate the information from the Markdown file and HARDCODE the new copy, hex colors, and image paths directly into the Astro components and CSS. Replace all existing Italian template defaults.
   - **`src/styles/global.css`**: Update the CSS variables in the `:root` block (e.g., `--primary`, `--bg-color`). If the Markdown file specifies a different typography style, update the `--heading-font` variable.
   - **`src/layouts/Layout.astro`**: Update the `<title>`, meta description, and the background gradient to match the new brand. Update the Google Fonts `<link>` if a new font is required.
   - **`src/components/Hero.astro`**: Rewrite the HTML content to use the headline, subtitle, and badge text from the Markdown file. Pick the most striking image from the `assets/images/` folder to serve as the Hero background, and update the `<img src="...">` path.
   - **`src/components/About.astro`**: Update the "Our Story" text and signature. Pick an image that best represents the restaurant's history or signature dish and update the `<img src="...">` path here.
   - **`src/components/Location.astro`**: Update the text describing the location and features. Pick an image representing the venue exterior or surroundings. (Note: if the research didn't provide location data, you can safely remove this section entirely from `src/pages/index.astro`).
   - **`src/components/Gallery.astro`**: Update the visual gallery (the `galleryItems` array) to map the most beautiful food/venue images from the `assets/images/` folder, choosing which images fit best to sponsor the venue. Provide corresponding titles and descriptions derived/inspired by the Markdown research.

3. **Assemble the Final Page (`src/pages/index.astro`)**
   - Open `src/pages/index.astro`.
   - Inject the actual contact information, business name, and hours from the Markdown file into the static HTML of the `<footer>`.

4. **Verify the Build**
   - Run `npm run build` to ensure there are no Astro compilation errors and all image paths are perfectly correct.
   - Report back to the user that the site has been successfully personalized!
